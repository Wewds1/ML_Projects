# CardShield Fraud Detection

CardShield is an end-to-end fraud detection project built from a dirty synthetic card-transactions dataset. The repo now includes the full analysis flow, a trained model artifact, a FastAPI backend, a non-Streamlit HTML/CSS/JavaScript frontend, Docker support, and saved EDA visuals for portfolio use.

## What is in here

- `data/raw/card_transactions.csv`: untouched source dataset
- `data/processed/cleaned_transactions.csv`: cleaned + engineered modeling dataset
- `01_eda/`: markdown writeups plus saved plots
- `models/`: model metrics, threshold config, and the serialized pipeline
- `src/pipeline.py`: cleaning logic, feature engineering, and preprocessing
- `scripts/train_pipeline.py`: reproducible training and artifact generation script
- `app.py`: FastAPI application
- `frontend/`: animated HTML/CSS/JS client served by FastAPI
- `Dockerfile`: container build targeting port `6000`
- `scratch1.md`: archived copy of the original project brief

## Project flow

The implementation follows the original brief:

1. Drop duplicate transaction rows while ignoring `transaction_id`.
2. Clean categorical inconsistencies in `merchant_category`, `channel`, and `device_type`.
3. Create missingness flags before imputation.
4. Impute:
   - `merchant_country` with the mode after saving a missing flag
   - `days_since_last_txn` with `999`
   - `avg_7day_spend` with a merchant-category median
5. Engineer risk-focused features such as:
   - `amount_to_limit_ratio`
   - `amount_vs_7day_avg`
   - `txn_velocity_ratio`
   - `merchant_concentration`
   - `foreign_online`
   - `late_night_foreign`
   - `new_account_high_amount`
6. Train multiple imbalance-aware models and save the best pipeline plus threshold config.

## Run locally

Install dependencies:

```bash
python -m pip install -r requirements.txt
```

Regenerate analysis, plots, and model artifacts:

```bash
python scripts/train_pipeline.py
```

Start the API and frontend locally:

```bash
python -m uvicorn app:app --host 0.0.0.0 --port 6000 --reload
```

Or on PowerShell:

```powershell
.\scripts\run_api.ps1
```

Then open [http://localhost:6000](http://localhost:6000).

## Docker

Build the container:

```bash
docker build -t cardshield-fraud .
```

Run it on port `6000`:

```bash
docker run -p 6000:6000 cardshield-fraud
```

## API

### `GET /health`

Returns a simple status payload.

### `POST /predict`

Accepts a transaction payload and returns:

- `fraud_probability`
- `is_fraud`
- `threshold_used`
- `action`
- `model_name`

Example payload:

```json
{
  "hour_of_day": 2,
  "day_of_week": 5,
  "is_weekend": 1,
  "month": 10,
  "cardholder_age": 32,
  "account_age_days": 41,
  "credit_limit": 4500,
  "merchant_category": "Online_Retail",
  "merchant_country": "US",
  "is_foreign": 0,
  "transaction_amount": 1299.99,
  "avg_7day_spend": 220.0,
  "num_txn_24h": 9,
  "num_txn_7d": 14,
  "days_since_last_txn": 0.2,
  "distinct_merchants_7d": 4,
  "declined_last_30d": 2,
  "channel": "Online",
  "device_type": "Mobile"
}
```

## Notes on structure

`src/` is intentionally light. It just holds the reusable project logic:

- `FraudFeatureBuilder`: cleans and engineers features the same way in training and inference
- `build_preprocessor()`: creates the numeric + categorical preprocessing stack
- `clean_transactions()`: produces the saved processed dataset for analysis

## Git hygiene

`.gitignore` excludes:

- virtual environments
- `__pycache__`
- `.pyc` files
- `.pkl` files
- `scratch1.md`

It does **not** ignore `README.md` or the generated PNG visual assets.

## Portfolio angle

This repo is now set up more like a small product than a notebook dump:

- reproducible training script
- saved visuals
- documented API
- animated frontend
- Dockerized runtime
- explicit threshold config

The dataset is synthetic, but the workflow is shaped around a realistic fraud-detection problem: heavy class imbalance, missingness as signal, threshold tradeoffs, and deployment-ready scoring.
